# Web Changelog v3.8.1 - MFA Returning User Flow

**Release Date:** 2025-12-14

## Changes

### Login Page
- Returning users with MFA now see verification options (Discord DM or Auth App)
- TOTP verification uses stored secret from database (no QR re-scan needed)
- Added `verifyTOTP` integration for login flow

### API Routes
- `/api/mfa/totp/setup` - Proxies to Bot API for encrypted storage
- `/api/mfa/totp/verify` - Supports setup and login verification modes

### OAuth
- Callback correctly passes `mfaMethods` array to frontend for returning users
